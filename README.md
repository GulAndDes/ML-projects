## Repository Structure Overview

Репозиторий отражает постепенную эволюцию от учебных экспериментов к более сложным прикладным системам в области машинного обучения и алгоритмической торговли.

Структура проектов выстроена по приоритету и уровню сложности.

---

## Алготрейдинг (ключевое направление)

Наиболее проработанная и технически сложная часть репозитория.

Содержит проекты, связанные с применением ML в финансовых рынках:

- построение торговых моделей
- работа с временными рядами (time series)
- walk-forward валидация
- калибровка вероятностей
- риск-менеджмент и управление капиталом

Это основной фокус репозитория и результат перехода от учебных задач к практическим ML-системам.

---

## Анализ дефектов (второй по значимости блок)

Проекты, посвященные анализу данных и задачам классификации, включая:

- выявление дефектов / аномалий в данных
- работа с несбалансированными выборками
- построение интерпретируемых моделей
- анализ качества предсказаний

Данный блок отражает опыт решения прикладных задач вне финансового домена и развитие навыков работы с реальными датасетами.

---

## Machine Learning (базовые проекты)

Набор учебных и экспериментальных проектов, с которых начиналось изучение ML.

Основные направления:

- базовая классификация и регрессия  
- EDA (анализ данных)  
- feature engineering  
- обучение моделей из библиотеки scikit-learn  

Эти проекты сыграли роль фундаментальной базы:

- сформировано понимание ML-пайплайна  
- освоены базовые алгоритмы  
- получен первый опыт работы с данными  

---

## Общая логика развития

Репозиторий отражает классический путь развития ML-инженера:

1. Учебные проекты → понимание основ  
2. Прикладные задачи → работа с реальными данными  
3. Сложные системы → end-to-end решения
